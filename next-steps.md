### Architecture Summary

```
IDEA PIPELINE (/refine)
├── Input: docs/initial-planning/
│   ├── full-idea.md (big picture)
│   ├── scope.md (Pareto focus)
│   └── manual.md (non-automated tasks)
├── Phase 1: Discovery questions
├── Phase 2: Validate/enhance initial planning
└── Phase 3: PRD + Architecture → docs/finalized-plan/

EXECUTION PIPELINE (/orc-exe)
├── Prerequisites: prd.md + architecture.md must exist
├── Story Generation: SM creates stories + parallel chunks
├── First Pass: Skeleton (functional, ugly OK)
│   └── Checkpoint: Notification + /clear recommendation
├── Second Pass: Polish (beautiful, Playwright validated)
│   └── Checkpoint: Notification + /clear recommendation
└── Third Pass: Fix (bugs, edge cases, production ready)
    └── Checkpoint: sprint_complete notification

SESSION CONTINUITY
├── /handoff → Saves state, pushes to GitHub
└── /resume → Reads state, continues from any device

STANDARDS
├── docs/standards/ui-ux-guide.md - UI patterns
└── docs/standards/infrastructure.md - Tech setup
```
